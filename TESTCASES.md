### E2E

## TC-E2E-01 Полный жизненный цикл объявления: от создания до проверки статистики

**Предусловия:**
1. Выбран уникальный sellerID (например, 111289).

**Шаги:**
1. Создать объявление (POST /api/1/item)
```json
{
  "sellerID": 111289,
  "name": "test",
  "price": 15000,
  "statistics": {
    "likes": 5,
    "viewCount": 12,
    "contacts": 10
  }
}
```

  - **Ожидаемый результат:** 
    - 200 OK, в ответе получен id (сохраняем его для следующих шагов).\

2. Запросить созданное объявление по ID (GET /api/1/item/{id})
   
  - **Ожидаемый результат:** 
    - 200 OK, данные в теле ответа (name, price, sellerID,statistics) полностью соответствуют отправленным в шаге 1.

3. Проверить наличие объявления в списке продавца (GET /api/1/111289/item)
   
  - **Ожидаемый результат:** 
    - 200 OK, в массиве объявлений присутствует объект с id из шага 1.

4. Проверить корректность статистики (GET /api/1/statistic/{id})
   
  - **Ожидаемый результат:** 
    - 200 OK, значения likes, viewCount, contacts соответствуют данным, переданным при создании в шаге 1.

**Общий ожидаемый результат:** 
  - Данные консистентны на всех этапах, id связывает все сущности корректно.
