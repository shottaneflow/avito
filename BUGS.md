# BUGS.md — Найденные баги

## BUG-01: POST /api/1/item некорректное тело ответа

**Severity:** Critical  
**Тест-кейс:** TC-POST-01  

**Ожидаемый результат** (согласно Postman-коллекции):
```json
{
  "id": "<string>",
  "sellerId": "<integer>",
  "name": "<string>",
  "price": "<integer>",
  "statistics": {
    "likes": "<integer>",
    "viewCount": "<integer>",
    "contacts": "<integer>"
  },
  "createdAt": "<string>"
}
```

**Фактический результат:**
```json
{
  "status": "Сохранили объявление - 94efaede-0f91-4502-9218-00bfe3f2eca3"
}
```

**Описание:** Сервер возвращает только строку со статусом и id внутри неё. 

---

## BUG-02: POST /api/1/item отрицательный price принимается

**Severity:** Major  
**Тест-кейс:** TC-POST-08  

**Шаги воспроизведения:**
```json
{
  "sellerID": 111289,
  "name": "test",
  "price": -1,
  "statistics": { "likes": 10, "viewCount": 10, "contacts": 10 }
}
```

**Ожидаемый результат:** 400 Bad Request  
**Фактический результат:** 200 OK, объявление создаётся с отрицательной ценой  

**Описание:** Цена товара не может быть отрицательной.

---

## BUG-03: POST /api/1/item — отрицательные значения в statistics принимаются

**Severity:** Major  
**Тест-кейс:** TC-POST-09  

**Шаги воспроизведения:**
```json
{
  "sellerID": 111289,
  "name": "test",
  "price": 100,
  "statistics": { "likes": -1, "viewCount": -1, "contacts": -1 }
}
```

**Ожидаемый результат:** 400 Bad Request  
**Фактический результат:** 200 OK, объявление создаётся с отрицательной статистикой  

**Описание:** Поля `likes`, `viewCount`, `contacts` не имеют проверки на неотрицательность. 

---

## BUG-04: POST /api/1/item создание объявления без поля statistics

**Severity:** Major  
**Тест-кейс:** Дополнительный  

**Шаги воспроизведения:**
```json
{
  "sellerID": 111289,
  "name": "test",
  "price": 100
}
```

**Ожидаемый результат:** 200 OK  
**Фактический результат:** Запрос завершается с ошибкой / неожиданным поведением  

**Описание:** Поведение при отсутствии блока `statistics` не определено. 

---

## BUG-05: GET /api/1/item/{id} ответ возвращается как массив вместо объекта

**Severity:** Major  
**Тест-кейсы:** TC-GET-01, TC-GET-05  

**Ожидаемый результат**:
```json
{
  "id": "6eb9b015-...",
  "sellerId": 111289,
  ...
}
```

**Фактический результат:**
```json
[
  {
    "id": "6eb9b015-...",
    "sellerId": 111289,
    ...
  }
]
```

**Описание:** Эндпоинт принимает единственный `id` через path-параметр, что  подразумевает возврат одного объекта. Возврат массива противоречит REST-конвенции и усложняет клиентский код без причины.

---

## BUG-06: GET /api/1/statistic/{id} ответ возвращается как массив вместо объекта

**Severity:** Minor  
**Тест-кейс:** TC-STAT-01  

**Описание:** Аналогично BUG-05. Эндпоинт `/statistic/{id}` возвращает массив из одного элемента вместо объекта. Запрос по конкретному `id` всегда подразумевает единственный результат.

**Фактический результат:**
```json
[
  { "likes": 5, "viewCount": 12, "contacts": 8 }
]
```
